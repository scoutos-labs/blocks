Spec-first vs experiment-first (Draft 04 discipline)

  • Spec-first (design already forced): retry ([RNR-17] already gives idempotent retry — say it louder), env (keep the ban; document the secret-input pattern), per-node grant narrowing (pure algebra reuse), signed attestations (stay closed).
  • Experiment-first (paper design would be malpractice): N-of-M (record state machine — where both prior drafts' real bugs lived), fan-out, concurrency text, canonical-JSON hardening, headless oracle, Windows paths.
