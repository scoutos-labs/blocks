Blocks Skill Protocol — Draft 02: workflow composition + signed approvals

PRD · slug blocks-protocol-draft02 · 2026-07-02 · deliverables: PROTOCOL.md Draft 02 + reference implementation + dogfood example + zenbin republish · status: plan approved

Definition

Draft 02 adds two capabilities as one symmetry applied twice more — grants co-sign a block's capabilities (Draft 01); parents co-sign a child workflow's grants (composition); approvals co-sign an oracle's judgment (AAITL):

  • Composition: workflows declare typed outputs; a node may be workflow: "name@version" instead of block. Nesting is acyclic composition, not recursion. "Functions get return values."
  • Signed approvals: fuzzy contracts may require oracle.claims; answers must be Ed25519-signed by a workspace-registered key carrying those claims; signer-agnostic (human or agent). No PKI.

All changes are additive: a Draft-02 validator accepts every Draft-01 document. Spec and reference implementation land together; the consistency harness must stay green including a new check that re-verifies a real committed approval signature from the run document + registry using the spec's own formula.

Security analysis commitments (must appear in spec + tests)

  • Replay bound by domain prefix + workflowHash + runId + nodeId + input/answer digests; identical-context replay idempotent via [RNR-10].
  • Unauth DoS neutralized by no-attempt-increment-on-auth-failure. Tested.
  • Signature stripping via contract/run-doc edits = workspace-write actor; audited (blockHash shift; post-hoc verify breaks), stated per [PRM-7] honesty.
  • No expiry/revocation: declared limitation; registry file deletion + git history is the mechanism. No PKI.
  • Cross-protocol signature reuse blocked by domain prefix.
