Blocks runtime hardening

Make each run an immutable, auditable, capability-bounded ledger from creation through resume, deterministic execution, fuzzy recording, and post-run verification.

Definition

In Blocks, runtime hardening means the run document remains the source of truth: inputs are fixed at creation, a grant cannot be widened during execution, fuzzy pause-time inputs and hashes cannot be restamped, and audit recomputes evidence rather than trusting fresh flags or edited files.

It does not mean adding containers, a daemon, a model client, dynamic fan-out, a registry, or a broader gate language. network: false remains a reviewed declaration, not a sandbox.

Goals and constraints

  • Preserve the design law: the CLI runs deterministic nodes; an oracle enters only through record.
  • Deny filesystem capabilities unless both block permissions and workflow grants cover them.
  • Never persist, print, audit-log, or commit raw secret inputs or private keys.
  • Keep failures actionable: file/path or node, expectation, and fix.
  • Preserve unrelated working-tree changes and generated docs.

Implementation plan

1. Freeze run inputs and fuzzy pauses

Change: reject non-secret resume overrides; schema-parse and digest-check resumed secrets; retain persisted non-secrets; make repeated fuzzy pauses idempotent and refuse input/hash drift without mutation.

Acceptance:

  • Non-secret overrides, unknown resume inputs, invalid/mismatched secrets fail with exit 2 and leave state byte-identical.
  • Matching typed secrets resume successfully and remain absent from persisted plaintext.
  • A repeated unchanged fuzzy pause is stable; changed block/input fails without restamping.

Evidence: focused regression tests plus full npm test.

2. Enforce filesystem boundaries

Change: compute effective read/write globs; remove blanket workspace read; contain entry files; harden read/write blocks against symlink escape; use unpredictable mode-0600 temporary inputs with reliable cleanup.

Acceptance:

  • Entry access is limited to runtime necessities plus effective grants.
  • Outside reads/writes through symlinks are refused and no outside target changes.
  • exec.entry traversal/symlink escape is rejected.
  • Temporary files are mode 0600, outside tracked run artifacts, and removed after success/failure.

Evidence: Node permission-capable tests, fallback block-level fencing tests, no-leftover assertions.

3. Harden validation

Change: shape-check containers before iteration; validate identifiers and schema keyword types/regexes; reject malformed or empty argv and invalid placeholders.

Acceptance: malformed JSON documents return structured errors, never uncaught stack traces; all valid repository blocks/workflows remain valid.

Evidence: adversarial validator tests, workflow validation sweep, full suite.

4. Make planning static and non-mutating

Change: branch plan before runtime input resolution/state creation.

Acceptance: required-input workflows plan without values; plan --state needs no secret re-supply; neither mode creates or mutates run state.

Evidence: triage workflow plan transcript and before/after file/state assertions.

5. Add run audit UX

Change: add blocks audit <run.json> [--json] to verify protocol, workflow/block hashes, output contracts, workflow outputs, child runs, approvals/claims, and capabilities without exposing secrets.

Acceptance: current valid deterministic, fuzzy, signed/attested, and nested runs pass; tampered/missing/drifted artifacts produce named non-zero findings; JSON output is stable and secret-safe.

Evidence: positive/negative audit tests and CLI transcripts.

6. Refresh docs, examples, and CI

Change: update README/SPEC/runner/composer guidance; replace or label stale run examples; add Node CI for tests, workflow validation, static plan, and audits.

Acceptance: quickstarts match current versions and Draft 03 behavior; current examples audit from a clean checkout; CI command list passes; no secrets/private keys/runtime temp files are tracked.

Evidence: local CI-equivalent transcript, audit sweep, git diff/status review.
