# Legacy run examples

These files are historical artifacts from earlier Blocks drafts or from block/workflow versions that have since changed.

They are kept so readers can inspect old ledgers, but they are **not** current fixtures. Current `blocks audit` requires protocol-3 run semantics and current workflow/block hashes, so these legacy examples intentionally fail audit with findings such as `legacy-protocol` or hash drift.

Use `../current/` for committed runs that must audit cleanly.
